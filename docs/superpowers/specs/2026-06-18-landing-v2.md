## 1. 概述

### 1.1 一句话定义

升级 `GET /` landing page：加 3 个 social proof section（Hero Stats、Top 3 Headhunters、Latest 5 Placements）+ 全站视觉重做（动画、互动、品牌色）。

### 1.2 触发原因

v1 landing 是 functional MVP，但视觉平淡、缺少 social proof，访问者看不出来"这里有人、有岗、有成功"。

### 1.3 目标

1. 3 个新 social proof section 在 Hero 之后立刻出现（让访客秒看到平台有活力）
2. 视觉重做：品牌色 + 动画 + hover 互动
3. 保持 SSR HTML（无 SPA 框架依赖）

### 1.4 非目标

- 不做 real-time WebSocket（v1 是 snapshot）
- 不做 user testimonials / 头像（没有真实数据）
- 不做图表库（用纯 SVG / CSS）
- 不改 backend API

### 2.3 视觉细节

| 元素 | 效果 |
|------|------|
| **Hero background** | 渐变（teal-50 → white），底部有微妙的 SVG wave |
| **数字 counter** | JS CountUp，1.5s ease-out，从 0 滚到 target |
| **Card hover** | `transform: translateY(-4px); box-shadow: larger;` 0.2s |
| **Section fade-in** | IntersectionObserver 触发 `opacity 0→1; translateY(20px)→0` |

| **Section headers** | 加 accent bar（左侧 4px teal 实色长条）|

### 2.4 JS 加载策略

- JS 内联在 HTML 底部（无外部依赖）
- `<noscript>` 友好：纯 CSS 数字静态显示
- 渐进增强：没 JS 也能看完整内容

## 3. 数据来源

### 3.1 Hero Stats

```sql
-- 今日解锁数（recommendation.status='unlocked' AND updated_at 今日）
SELECT COUNT(*) FROM recommendations
WHERE status = 'unlocked' AND updated_at >= datetime('now', 'start of day');

-- 今日 placement 数（placements.status='placed' AND updated_at 今日）
SELECT COUNT(*) FROM placements
WHERE updated_at >= datetime('now', 'start of day');

-- 活跃候选人数（candidates_anonymized.total）
SELECT COUNT(*) FROM candidates_anonymized;

-- API uptime（time since startup / hardcoded 99% if no data）
-- 简单方案：显示静态 "99.9%"，未来接 metrics
```

### 3.2 Top 3 Headhunters

复用 `/v1/market/leaderboard` 查询，limit 3。

```sql
SELECT id, name, reputation FROM users
WHERE user_type = 'headhunter' AND status = 'active'
ORDER BY reputation DESC LIMIT 3;
```

### 3.3 Latest 5 Placements

```sql
-- placements JOIN jobs（拿 industry / title）
-- placements JOIN users（拿 headhunter name）
SELECT p.id, p.annual_salary, p.updated_at, j.industry, j.title, h.name as headhunter_name
FROM placements p
LEFT JOIN jobs j ON p.job_id = j.id
LEFT JOIN users h ON p.primary_headhunter_id = h.id
WHERE p.status = 'placed'
ORDER BY p.updated_at DESC
LIMIT 5;
```

**Privacy considerations**:
- 不显示 candidate_user_id / candidate name
- 显示 industry 而非真实公司名（job.industry 是平台定义的值）
- headhunter name 显示（headhunter 想要 visibility）

## 6. 错误处理

| 场景 | 行为 |
|------|------|
| 无 placements | "暂无最近 placement 记录" + CTA "成为第一个推荐成功的人" |
| 无 headhunters in leaderboard | 隐藏 Top 3 section |
| 数字 = 0 | 显示 0（不隐藏） |
| DB 查询失败 | hero stats 显示 "—" 而非数字 |
