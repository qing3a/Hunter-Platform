## What's New

### 1. 项目重新定位：API 优先
- `package.json` 从 `convo` 改名为 `hunter-platform`，description 从 "AI desktop client" 改为 "猎头中介 API 平台"
- `README.md` 完全重写，描述 API 优先 + Claude/Agent 自助接入流程
- `pnpm dev` 默认启动 API 模式（不再走 electron-vite）

- `src/main/index.ts` 加 mode banner，方便看启动时是 API-only 还是 Electron 模式

### 2. 渲染层（Render Layer）—— 一次性 token 视图系统

这是本版本最大的功能新增。

- **8 个新模块** + **4 个 SSR HTML 模板**（候选人画像 / 推荐状态时间线 / 用户配额 / 审计日志）
- **4 个新视图端点**：
  - `GET /view/candidate/{anon_id}` — 脱敏候选人画像
  - `GET /view/recommendation/{rec_id}` — 4 步解锁状态时间线
  - `GET /view/user-quota/{id}` — 配额 + 限流面板
  - `GET /view/audit/{id}` — 审计时间线
- **2 个 token 生成端点**（解决数组响应无法自动注入 view_url 的问题）：
  - `POST /v1/views/audit/{user_id}` — 用户自己拿自己的 audit view URL
  - `POST /v1/views/recommendation/{rec_id}` — 猎头/雇主拿推荐 view URL
- **自动注入**：业务 API 响应自动包含 `view_url` 字段（`route-view-map.ts` 配置驱动）
- **1 小时过期 + 单次性**：token 创建后 1 小时有效，第一次访问即标记消耗，第二次 410 Gone
- **零依赖**：用 Node 内置 `crypto.randomBytes`，HTML 模板用纯字符串插值（无 React/Vue/EJS）

### 3. API 字段命名约定（Convention A）

资源自我 ID 统一为 `id`，外键用 `<resource>_id`，AnonymizedCandidate 保留 `anonymized_id`（语义例外）。

约定文档写进 `docs/superpowers/skill.md` 第 2.5 节，作为后续 API 设计的规则。

### 4. 审计基础设施（action_history 中间件）

- 自动 PII 脱敏的请求/响应摘要
- 4 个业务路由前缀（auth/register、headhunter、employer、candidate）自动挂载
- 通过 `res.on('finish')` 延迟读取 `req.user`（解决中间件挂载顺序问题）
- 12 任务完整 plan 已写入 `docs/superpowers/plans/`

### 5. Desensitize 行业映射升级

- 200+ 公司 → 12 个行业分类的 JSON 数据文件
- `lookupIndustry()` 函数统一处理（替代散落的 INDUSTRY_MAP 直接查找）
- 完整的 enumeration + fallback 单元测试

## Breaking Changes

### `POST /v1/auth/register` 响应字段重命名

**Before**:
```json
{ "data": { "user_id": "user_xxx", "api_key": "hp_live_xxx", ... } }
```

**After**:
```json
{ "data": { "id": "user_xxx", "api_key": "hp_live_xxx", ... } }
```

**迁移指南**：所有 `data.user_id` 引用改为 `data.id`。`/v1/users/{user_id}/status` 和 `/v1/users/{user_id}/history` 等 URL 路径参数**不变**。

已添加 regression test 防止再次不一致：
- `tests/integration/register-naming.test.ts` 断言 `data.user_id` 为 undefined
- 断言 `data.id` 等于 `GET /v1/users/{id}/status` 的 `data.id`

## Migration Notes

**从 0.1.0 → 0.2.0**：

1. **必改**：替换 `data.user_id` → `data.id`（仅 register endpoint 响应）
2. **可选**：使用 view layer 的新视图端点提升可观测性
3. **环境变量**：`.env` 现有字段已足够，无新增

**给 Agent / API consumer 的建议**：
- 注册后用 `data.id` 调用后续 endpoint
- 拿到业务响应里的 `view_url` 时直接用浏览器打开即可，无需登录
- 需要 audit view 时调 `POST /v1/views/audit/{user_id}` 单独拿 token
- 需要 recommendation view 时调 `POST /v1/views/recommendation/{rec_id}`（限猎头/雇主）
