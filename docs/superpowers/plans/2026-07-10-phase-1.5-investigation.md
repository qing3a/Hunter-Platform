# Phase 1.5 Investigation — `/admin/login` React Mount Bug

**Date:** 2026-07-10
**Branch:** `5-spa-split/phase-1.5`
**Repo:** `D:\dev\hunter-platform`
**Scope:** Phase 1.5 only — investigate and fix the real React mount bug in `admin-web/`.

---

## Summary

The Phase 0 Playwright regression was reproducible on branch `5-spa-split/phase-1.5`: `/admin/login` returned HTTP 200 and loaded `/admin/src/main.tsx`, but React rendered nothing into `#root`.

Root cause: `AdminApp` was mounted under an outer `<Route path="/admin/*" ...>`, but its descendant `<Routes>` used absolute `/admin/...` paths. In this nested route context React Router did not match `/admin/login` to the login route, so `AdminApp` rendered no DOM. There was no exception, no failed request, and no redirect, which is why the blank page was silent.

Fix: make the admin sub-router routes relative to the outer `/admin/*` route and use an index route for `/admin`.

Result: the Phase 0 Playwright test now passes with `Root innerHTML length: 323`.

## Root cause

`D:\dev\hunter-platform\admin-web\src\App.tsx`: `AdminApp` was mounted under `/admin/*`, but its descendant routes were defined as absolute `/admin/...` routes; React Router matched the outer route but no inner admin child route, so `/admin/login` rendered an empty route tree instead of `LoginPage`.

---

## The fix

### `D:\dev\hunter-platform\admin-web\src\App.tsx` lines 78-104

Changed `AdminApp` child routes from absolute `/admin/...` paths to paths relative to the outer `/admin/*` route:

- `/admin/login` → `login`
- `/admin` → index route
- `/admin/users` → `users`
- `/admin/jobs/:id` → `jobs/:id`
- `/admin/*` → `*`

This fixes `/admin/login` and prevents the same descendant-route mismatch for other admin pages.

### `D:\dev\hunter-platform\admin-web\tests\e2e\admin-login.spec.ts` lines 12-128

Enhanced the Phase 0 Playwright diagnostic to capture `pageerror`, `unhandledrejection`, final URL, console warnings, `page.content()` tail, and `document.body.outerHTML` tail while keeping the hard root assertion. Also guarded the Playwright-only test from Vitest collection so `pnpm test` can still run the unit suite.
