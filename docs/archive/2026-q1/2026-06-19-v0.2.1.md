## Migration Notes

**从 v0.2.0 → v0.2.1**：无 API breaking change。

- 5 个新 endpoint 都遵循 Convention A：`self.id` / `xxx_id` FK
- `required_skills` 是**纯加法**字段，旧调用不受影响
- `browseTalent` 响应去掉了 `id` 字段（如果客户端依赖此字段需改用 `anonymized_id`）

**给 Agent / API consumer 的建议**：
- 候选人相关逻辑用 `anonymized_id`（一直如此）
- 调 `POST /v1/employer/jobs` 时可以传 `required_skills` 数组（可选）
- 想看 leaderboard / config 直接 hit 新 endpoint 即可

## Architecture Decision Records
