## What's New

### 1. Reference Agent (`examples/reference-agent/`)

自动验证 skill.md 列出的全部 **27 个 endpoint** 都能按文档工作（实际跑通 37 个 endpoint，因为多角色测试自然重复）。

```
npx tsx examples/reference-agent/src/index.ts
```

输出 `37/37 passed` 或带失败清单的详细报告。**作用**：
- Contract test：保证文档 = 实现
- 参考实现：给真实用户当调用范例
- 烟雾测试：CI 用

### 2. skill.md 文档补全

修复了 4 个之前漏掉的文档缺口（mimo 通过跑 reference agent 暴露的）：

| # | 修复 | 详情 |
|---|------|------|
| 1 | **`POST /v1/employer/placements` schema** | 加了真实 schema（用 `anonymized_candidate_id` 而非 `candidate_user_id`）|
| 2 | **同猎头重复推荐 → 409** | 加了说明：同一猎头对同一 (候选人, 岗位) 推荐两次会返回 `DUPLICATE_REQUEST` |
| 3 | **IP 级别 register 限流** | 加了说明：单 IP 5 次/小时；绕过方法（`DELETE FROM rate_limit_buckets`）|
| 4 | **Config / Market 鉴权** | 表格加了"鉴权"列，明确 `/v1/config/*` 和 `/v1/market/*` 需要 Bearer |

### 3. 新增 2 个 endpoint（之前未列在 skill.md）

- `POST /v1/views/audit/{user_id}`（v2 render layer）
- `POST /v1/views/recommendation/{rec_id}`（v2 render layer）

skill.md 现在覆盖 **27 个 endpoint**。

## 项目最终能力

### 三角色 API（27 endpoints）
- **Auth**: register
- **User**: status + history (per role)
- **Employer**: jobs, talent browse, recommendations, unlock, placements
- **Headhunter**: candidates upload + publish, recommendations, withdraw
- **Candidate**: opportunities, access-log, approve/reject unlock, GDPR delete
- **Market/Config**: leaderboard, industries, title_levels, salary_bands
- **Public**: health, skill.md, openapi.json, metrics
- **View layer** (render tokens): audit + recommendation views

### 业务能力
- 候选人 PII AES-256-GCM 加密 + 字段级脱敏
- 4 步解锁状态机（pending → employer_interested → candidate_approved → unlocked → placed）
- 配额 / 限流 / 幂等 / webhook 异步投递
- 审计日志（action_history）
- 一次性 token 视图（5 种类型）
- 公开 landing 页（marketplace 风格 + social proof）

### 工程能力
- 5 个 DB migration
- TypeScript strict mode + ESM
- 零外部运行时依赖（5 个）
- 完整 superpowers 流程（brainstorm → spec → plan → execute → release notes）
- Reference agent 自动验证 docs vs code

## What It Looks Like to a User

一个外部 Agent 想集成 Hunter Platform：

1. 读 `https://.../v1/skill.md`（27 endpoint 完整文档 + schema）
2. 调 `POST /v1/auth/register` 拿到 `api_key`
3. 调各种业务 endpoint
4. 收到响应里的 `view_url`（一次性 token）→ 浏览器打开看人类可读的 HTML
5. 业务完成

整个过程 **不需要任何 UI、任何 SDK、任何认证流程**（除了 Bearer token）。

## 这是不是 1.0？

**是**——任何能调 HTTP 的 Agent 现在可以按 skill.md 完整集成。项目达到了"文档准确、实现完整、可被外部使用"的最低标准。
